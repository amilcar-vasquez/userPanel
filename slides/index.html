cloud Docker Overview

  • Definition: Docker is an open-source platform that simplifies building, shipping, and running applications using containers.
  • Problem solved: Ensures environment consistency across developer laptops, test servers, and production by packaging an app and its dependencies together.
  • Why containers vs VMs: Containers share the host OS kernel, making them lightweight, efficient, and fast to start compared to full virtual machines.
  • Key objects:
    • Dockerfile — a scripted set of instructions used to build a Docker Image.
    • Docker Image — a read-only template (artifact) produced from a Dockerfile.
    • Docker Container — a live, isolated, runnable instance created from an image.

devices Docker: Client–Server Architecture

  • Architecture: Docker uses a client–server model — the Docker Client (CLI) talks to the Docker Daemon (dockerd) running on the host.
  • Daemon role: The Daemon manages Docker objects: images, containers, networks, and volumes (persistent storage).
  • Standard workflow: Define environment in a Dockerfile → Daemon builds an immutable Docker Image → push image to a Registry (e.g., Docker Hub) → run a Container from the image in target environment.
  • Version note: Docker Engine community releases are commonly in the 24.0+ series; consult the official docs for the latest stable download per OS.
  • Why it matters: This model centralizes build/run operations and makes deployments reproducible, portable, and consistent across environments.

lightbulb Benefits, Use Cases, and Application

  • Portability & Consistency: Eliminates environment-related issues by packaging apps with their dependencies, ensuring identical behavior across laptop, test, and production.
  • Faster Deployments & Cost Efficiency: Containers start quickly and use resources more efficiently than VMs, enabling rapid deployment cycles and lower infrastructure costs.
  • Microservices Foundation: Ideal for microservices — apps can be split into smaller, independently deployable units.
  • DevOps & CI/CD: Standardized container images simplify CI/CD pipelines so the code tested is the same code deployed.
  • Reliability: Containers promote reproducible builds and predictable runtime behavior, accelerating iteration and reducing debugging time.

security Container Security: Best Practices

  • Minimal & Trusted Images: Use minimal base images (e.g., Alpine) from trusted sources to reduce attack surface.
  • Image Scanning: Scan images for known CVEs before deployment (tools: Trivy, Docker Scan, Clair).
  • Least Privilege at Runtime: Run containers as a non-root user and restrict Linux capabilities.
  • Secrets Management: Do not bake secrets into images; inject them at runtime via secrets managers or environment variables stored securely.
  • Runtime Hardening: Limit container access to host resources, use read-only filesystems where possible, and enable resource limits (CPU/memory).

cloud Docker: Install & Quick Setup

  • What: Docker provides container runtime; Docker Compose manages multi-service stacks.
  • Windows / macOS: Install Docker Desktop from https://docs.docker.com/desktop/. Enable WSL2 on Windows if prompted.
  • Linux (Ubuntu/Debian): install via package manager: 
    sudo apt update
sudo apt install -y docker.io docker-compose
sudo usermod -aG docker $USER
  • Quick start (repo root): 
    docker compose up --build -d

# view logs
docker compose logs -f

# stop and remove
docker compose down
  • Notes: Use a .env or backend/.env to set secrets (e.g., JWT_SECRET); volume db_data persists Postgres data; service healthchecks control startup order.

language Frontend: API Integration

// lib/github.ts
export interface GitHubProfileResponse {
    profile: GitHubProfileStats;
    rank: RankInfo;
}

export async function fetchGithubProfile(): 
    Promise<GitHubProfileResponse> {
    const response = await get<GitHubProfileResponse>(
        '/github/profile'
    );
    if (response.success && response.data) {
        return response.data;
    }
    throw new Error(response.message || 'Failed to fetch profile');
}

    build: ./frontend
    ports: ["5173:5173"]
    depends_on: [api]
